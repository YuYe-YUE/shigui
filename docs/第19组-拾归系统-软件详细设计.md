# 第19组 拾归系统 — 软件详细设计

> 本文档随开发进度持续更新。当前版本：Sprint 8 完成（2026-05-18）。

## 1. 数据库设计

10 张表 + post_image，全部使用逻辑删除 (`deleted TINYINT DEFAULT 0`)，`status` 只表示业务流转。LostFoundPost 不使用 `@TableLogic`（管理员需查看已删单据）。

### 1.1 app_user — 小程序用户

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK AUTO_INCREMENT | 主键 |
| openid | VARCHAR(64) UNIQUE NOT NULL | 微信 OpenID |
| nickname | VARCHAR(64) DEFAULT '' | 昵称 |
| avatar_url | VARCHAR(512) DEFAULT '' | 头像 URL |
| role | VARCHAR(16) DEFAULT 'USER' | 角色 |
| status | VARCHAR(16) DEFAULT 'NORMAL' | NORMAL / BANNED |
| deleted | TINYINT DEFAULT 0 | 逻辑删除 |

### 1.2 admin_user — 后台管理员

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK AUTO_INCREMENT | 主键 |
| username | VARCHAR(64) UNIQUE NOT NULL | 用户名 |
| password_hash | VARCHAR(256) NOT NULL | `salt:SHA256(salt+SHA256(password))`，前端先 SHA-256 再发送，避免明文传输 |

### 1.3 lost_found_post — 失物招领单据

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGINT PK AUTO_INCREMENT | 主键 |
| user_id | BIGINT NOT NULL | 发布者 ID |
| post_type | VARCHAR(8) NOT NULL | LOST / FOUND |
| title | VARCHAR(128) | 标题 |
| item_name | VARCHAR(64) | 物品名称 |
| item_category | VARCHAR(32) | 品类 |
| description | TEXT | 描述 |
| private_feature | TEXT | 非公开特征（认领核验用） |
| campus_area | VARCHAR(64) | 校区 |
| location_name | VARCHAR(128) | 具体地点 |
| longitude | DECIMAL(10,7) | 经度 |
| latitude | DECIMAL(10,7) | 纬度 |
| storage_location | VARCHAR(256) | 暂存地点（仅招领单） |
| event_time | DATETIME | 发生时间 |
| status | VARCHAR(32) DEFAULT 'PENDING_AUDIT' | PENDING_AUDIT / MATCHING / CLAIMING / RETURNING / COMPLETED |
| published_at | DATETIME | 发布时间 |
| deleted | TINYINT DEFAULT 0 | 逻辑删除 |

### 1.4 其余 7 张表

`audit_record`（审核日志, action=APPROVE/DELETE）、`claim_record`（认领）、`match_record`（匹配）、`notification`（通知）、`chat_session`（聊天会话）、`chat_message`（聊天消息）、`system_config`（系统配置）。完整 DDL 见 `scripts/init_schema.sql`。

## 2. API 设计

统一响应体 `Result<T>`：`{code, message, data}`。鉴权方式：请求头 `satoken: <token>`。

### 2.1 用户接口

| 端点 | 鉴权 | 说明 |
|------|------|------|
| `POST /api/user/wx-login` | 公开 | body: `{openid}` → `{code:200, data:"<token>"}` |
| `GET /api/user/me` | 登录 | 返回当前用户完整信息 |

### 2.2 管理员接口

| 端点 | 鉴权 | 说明 |
|------|------|------|
| `POST /api/admin/login` | 公开 | body: `{username, password}`（password 为前端 SHA-256 哈希值）→ token |

### 2.3 单据接口

| 端点 | 鉴权 | 说明 |
|------|------|------|
| `POST /api/posts` | 登录 | 发布单据, body 见 CreatePostRequest |
| `GET /api/posts` | 公开 | 列表（仅 MATCHING 状态）。?page=&size=&postType=&itemCategory=&campusArea=&keyword= |
| `GET /api/posts/mine` | 登录 | 我的记录。?page=&size=&postType= |
| `GET /api/posts/{id}` | 公开 | 单据详情。登录用户且为本人时返回联系方式 |
| `GET /api/posts/map` | 公开 | 地图点位（仅 FOUND + MATCHING + 有坐标） |

CreatePostRequest 字段：`postType, title, itemName, itemCategory, description, privateFeature, campusArea, locationName, longitude, latitude, storageLocation, eventTime, imageUrls`。必填：title, itemName, itemCategory, campusArea, locationName, eventTime。postType 必须为 LOST 或 FOUND。imageUrls 最多 3 个且必须以 `/uploads/posts/` 开头。

PostResponse 字段：`id, postType, title, itemName, itemCategory, description, campusArea, locationName, storageLocation, eventTime, publishedAt, status, coverImageUrl, imageUrls`。不含 privateFeature。

### 2.4 匹配与通知接口

| 端点 | 鉴权 | 说明 |
|------|------|------|
| `GET /api/matches/mine?page=&size=` | 登录 | 我的匹配记录 |
| `GET /api/notifications?page=&size=` | 登录 | 我的匹配提醒 |

匹配流程：审核通过 → `AdminPostService.approvePost()` → `MatchRecordService.generateMatchesForPost()` → 规则预筛 + DeepSeek v4-flash 语义排序 → 去重去阈值 → 写 `match_record` + 双方 `notification`。AI 调用失败时规则兜底。`threshold`/`maxCandidates`/`maxResults` 通过 `AiMatchProperties` 配置。

### 2.5 认领接口

| 端点 | 鉴权 | 说明 |
|------|------|------|
| `POST /api/claims` | 登录 | 发起认领, body: `{postId, privateFeatureAnswer}` |
| `GET /api/claims/mine?page=&size=` | 登录 | 我的认领记录 |
| `GET /api/claims/{id}` | 登录 | 认领详情（校验所有权：仅申请人可查看） |
| `PUT /api/claims/{id}/confirm-receive` | 登录 | 确认收到（校验所有权） |

认领状态流转：`PENDING_AI_REVIEW` → AI 预审 → `VERIFIED` / `REJECTED` / `PENDING_ADMIN_REVIEW` → 管理员审核 → `COMPLETED`。`createClaim()` 使用 `@Transactional`。

### 2.6 聊天接口

| 端点 | 鉴权 | 说明 |
|------|------|------|
| `POST /api/chat/sessions` | 登录 | 创建/复用会话, body: `{postId}` |
| `GET /api/chat/sessions/{id}/messages` | 登录 | 消息列表（校验参与者身份） |
| `POST /api/chat/sessions/{id}/messages` | 登录 | 发消息, body: `{content}`（校验参与者身份） |

角色脱敏：消息发送者仅暴露 `LOST`/`FOUND` 角色，不暴露真实身份。`requireParticipantSession()` 校验当前用户是否为会话的 `lostUserId` 或 `foundUserId`。

### 2.7 文件上传接口

| 端点 | 鉴权 | 说明 |
|------|------|------|
| `POST /api/files/upload` | 登录 | 上传图片, multipart `file` → `{url}` |

仅支持 JPG/PNG/WebP，最大 5MB，校验图片内容有效性，文件按日期目录存储（`uploads/posts/YYYY/MM/DD/UUID.ext`），防路径穿越。

### 2.8 管理端接口

| 端点 | 鉴权 | 说明 |
|------|------|------|
| `POST /api/admin/login` | 公开 | 管理员登录（password 为前端 SHA-256 哈希值） |
| `GET /api/admin/posts?page=&size=&status=` | 管理员 | 全量单据列表，列表不含 privateFeature |
| `GET /api/admin/posts/{id}` | 管理员 | 单据详情含 privateFeature |
| `POST /api/admin/posts/{id}/approve` | 管理员 | 审核通过 → MATCHING，触发 AI 匹配 |
| `DELETE /api/admin/posts/{id}` | 管理员 | 删除单据（reason 必填） |
| `GET /api/admin/dashboard` | 管理员 | 仪表盘统计（注册用户/匹配中/今日发布/成功认领/匹配记录数） |
| `GET /api/admin/matches?page=&size=` | 管理员 | 所有匹配记录列表 |
| `GET /api/admin/claims?page=&size=&status=` | 管理员 | 认领列表 |
| `PUT /api/admin/claims/{id}/approve` | 管理员 | 认领审核通过 |
| `PUT /api/admin/claims/{id}/reject` | 管理员 | 认领审核拒绝, body: `{reason}` |
| `GET /api/admin/users?page=&size=&status=` | 管理员 | 用户列表 |
| `PUT /api/admin/users/{id}/ban` | 管理员 | 封禁用户 |
| `PUT /api/admin/users/{id}/unban` | 管理员 | 解封用户 |

管理员鉴权：`StpUtil.login(adminId + 10_000_000)`，`requireAdmin()` 校验 `loginId >= 10_000_000`，普通用户 token 返回 403。审核通过/删除由 `AdminPostService` 做 `@Transactional` 原子操作。

### 2.9 权限控制

- **详情权限**：`GET /api/posts/{id}` 非本人只能查看 `status=MATCHING` 的公开单据
- **认领权限**：`getByIdOrThrow` 校验 `claimantUserId`，`confirmReceive` 校验所有权
- **聊天权限**：`requireParticipantSession()` 校验当前用户属于会话参与者
- **管理员权限**：`requireAdmin()` 校验 `loginId >= 10_000_000`

### 2.10 Sa-Token 鉴权配置

拦截 `/api/**`，排除：`/api/user/wx-login`, `/api/admin/login`, `/api/posts/map`, `/api/posts/**`。POST/mine 在 Controller 内手动验登录。`NotPermissionException` → 403, `NotLoginException` → 401。

## 4. 前端架构

### 4.1 小程序

原生微信小程序框架，11 页面 + post-card 组件 + 20 个图标。中大绿（#00573D）杂志卡片风。app.globalData 管理 token 和 baseUrl。

### 4.2 管理端

Vue 3 + Vite + TypeScript + Element Plus + Pinia + Vue Router + Axios。左侧导航 + 右侧内容经典后台布局。

- **API 代理**：开发时 Vite 代理 `/api` 和 `/uploads` → `http://localhost:8080`，部署时 `baseURL: ''` 使用相对路径由 Nginx 统一转发
- **登录安全**：前端用 Web Crypto API 对密码做 SHA-256 哈希后再发送，避免明文出现在请求体和网络日志中
- **鉴权**：Axios 请求拦截器注入 `satoken` 请求头，401 响应自动跳转登录页
