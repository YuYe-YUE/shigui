# 第19组 拾归系统 — 用户手册

> 随开发进度持续更新。当前版本：Sprint 5.5 完成（2026-05-14）。

## 1. 系统简介

拾归是中山大学校园失物招领平台。失主发布寻物启事，拾捡者发布招领信息，系统自动进行智能匹配并通知双方。校方保卫处通过 Web 后台进行内容审核和用户管理。

- **用户端**：微信小程序
- **管理端**：Web 浏览器

## 2. 小程序使用说明

### 2.1 登录

1. 打开小程序，底部导航切换到"我的"
2. 点击"微信一键登录"按钮
3. 完成后显示用户信息

### 2.2 浏览失物招领

1. 打开"首页" tab，查看瀑布流列表
2. 点击顶部胶囊切换"寻物/招领"
3. 横向滑动选择品类（校园卡/学生证/钥匙/耳机/水杯/雨伞/书籍/其他）
4. 在搜索框输入关键词搜索标题
5. 下拉刷新，触底自动加载更多
6. 点击卡片进入详情页
7. 点击右上角地图图标查看地图分布

### 2.3 发布寻物启事

1. 切换到"发布" tab
2. 点击"发布寻物启事"卡片
3. 填写物品名称、类别、时间、校区、地点等必填信息
4. 可选填写描述和私密特征（私密特征仅用于认领核验，不会公开）
5. 点击"提交发布"
6. 提交后进入待审核状态，管理员审核通过后展示在首页

### 2.4 发布招领启事

1. 切换到"发布" tab
2. 点击"发布招领启事"卡片
3. 与寻物启事相同，额外填写暂存地点
4. 提交后等待审核

### 2.5 查看我的记录

1. 切换到"我的" tab
2. 点击"我的寻物"或"我的招领"进入独立页面查看已发布单据
3. 支持下拉刷新和触底翻页

### 2.6 查看匹配结果

1. 切换到"我的" tab
2. 点击"我的匹配"查看 AI 智能匹配结果
3. 点击"匹配提醒"查看系统推送的匹配通知
4. 点击卡片可跳转对应单据详情

## 3. 管理端使用说明

### 3.1 登录

1. 浏览器打开 `http://localhost:5173/login`
2. 输入管理员账号密码（开发环境：admin / admin123）
3. 登录后进入仪表盘

### 3.2 内容审核

1. 左侧菜单点击"内容审核"
2. 切换"待审核"/"全部" Tab 查看单据
3. 点击"详情"查看物品描述、私密特征、暂存地点
4. 点击"通过"审核通过，单据进入匹配池
5. 点击"删除"→ 确认 → 填写删除原因 → 删除

### 3.3 用户管理

1. 左侧菜单点击"用户管理"
2. 查看小程序用户列表（昵称、状态、注册时间）
3. 点击"封禁"/"解封"管理用户权限

### 3.4 查看匹配结果

1. 左侧菜单点击"匹配结果"
2. 查看所有 AI 匹配记录（失物单、招领单、匹配度、原因）
3. 点击按钮查看对应单据详情

## 4. 开发环境部署

### 4.1 前提

- Java 21, Maven, Node.js, MySQL 8.0
- 微信开发者工具

### 4.2 数据库初始化

```bash
mysql -u root -p < scripts/init_schema.sql
mysql -u root -p < scripts/seed_data.sql
```

### 4.3 启动后端

IDEA 打开 `backend/`，运行 `BackendApplication.java`，端口 8080。

### 4.4 启动管理端

```bash
cd admin-web && npm install && npm run dev
```

浏览器访问 `http://localhost:5173`。

### 4.5 启动小程序

微信开发者工具打开 `miniapp/` 目录。注意关闭域名校验（开发工具 → 详情 → 不校验合法域名）。
