### 2.3 关键业务场景覆盖

- 微信登录（新用户自动创建、已有用户返回 ID）
- 密码校验（SHA-256 + salt，正确/错误/不存在）
- 单据发布（参数校验逐字段、封禁用户拦截、状态=PENDING_AUDIT）
- 列表查询（MATCHING 过滤、deleted=0 过滤、postType/品类/校区/关键词组合筛选）
- 我的记录（用户隔离、不限状态、排除已删除）
- 内容审核（列表/详情、审核通过状态校验、删除原因必填、deleted=1 不可重复操作）
- 用户管理（列表/封禁/解封、NORMAL/BANNED 状态流转）
- 管理员鉴权（requireAdmin 校验、普通用户调 admin API 返回 403、NotPermissionException Handler）
- 详情权限管控（非本人不能查看 PENDING_AUDIT 单据）
- 审核事务（AdminPostService @Transactional，状态变更+审计日志原子操作）
- AdminWeb 路由守卫（无 token 自动跳转登录页）
- 管理员越权防护（approve/delete/ban 普通用户 token → 403）
- 列表数据脱敏（privateFeature 不在 PostResponse 中）
- 登录失败不跳转（auth.login 检查 code 后才存 token）
- 已删单据管理端可视（LostFoundPost 取消 @TableLogic，公开接口显式过滤）
- 公开浏览链路（/api/posts/** 排除 SaToken，POST/mine 手动验登录）
- AI 匹配（DeepSeek v4-flash 语义排序 → 规则兜底、AiMatchProperties 配置化、私密特征脱敏）
- 匹配分页（数据库层按用户 postId 过滤，不过滤于内存）
- AdminWeb Axios 拦截器 reject 非 200（避免业务失败仍继续执行）
- AdminWeb 加载状态（PostAuditView/UserManageView/MatchResultView v-loading + try/catch）
- 小程序"我的"页（10 个入口全部贯通，我的寻物/招领 → 独立二级页 my-posts）

```bash
cd backend && ./mvnw test
```
